# 小程序档案页多人档案设计

**背景**

当前小程序端 `档案` 页只有单人、单档案、单空状态展示，页面把“档案内容”和“预约入口”耦合在一起，导致以下问题：

- 不支持绑定多位成员档案
- 不支持同一成员存在 0 次、1 次、多次就诊记录的差异状态
- 当前空状态下才出现 `去预约`，不符合用户“预约入口始终存在”的预期
- 多次就诊记录缺少明确的查看入口，用户无法快速定位每次记录详情

本次需求是在保留当前小程序视觉语言的前提下，把 `档案` 页扩展成支持多人档案、多次记录和新增成员入口的完整原型页。

**目标**

- 在小程序 `档案` 页支持展示多个已绑定成员档案
- 至少覆盖 3 种成员状态：
  - 成员 A：2 次就诊记录
  - 成员 B：1 次就诊记录
  - 成员 C：已创建档案但 0 次记录，可从新增入口进入
- `去预约` 按钮始终固定在页面底部，不与记录是否为空绑定
- 同一成员下的每次就诊记录以独立子卡片展示，点击后向下展开详情
- 展开详情区复用当前档案页已有的 `左右眼 + 眼轴 / 屈光度` 卡片语义，不另起一套视觉体系

**确认方案**

用户已确认采用 `A1` 方案：

- 页面上方先切换成员档案，而不是同时平铺多个完整档案
- 当前只展示 1 位成员的主档案卡片，避免页面高度失控
- 主档案卡片内部展示多条“就诊记录子卡片”
- 任意一条记录点击后，在该条子卡片内部向下展开详情
- 页面底部保留固定预约浮层，成员无记录时也可以直接预约
- 页面保留一个明显的 `新增成员档案` 入口，用于创建第 3 位成员的 `0 次记录` 状态

**页面结构**

1. 顶部固定区域

- 保留小程序壳层、顶部操作区和 `问诊 / 档案 / 工具 / 智能体` 标签结构
- `档案` 仍为当前激活标签
- 保留当前页已有的吉祥物展示区，用于延续既有视觉记忆

2. 成员切换条

- 位于吉祥物下方，横向排列
- 至少包含 3 个入口：
  - `周末 2次`
  - `安安 1次`
  - `+ 新增`
- 当前激活成员使用蓝色渐变高亮，未激活成员使用白底浅描边卡片
- 点击不同成员后，主档案卡片整体切换到对应成员数据

3. 主档案卡片

- 延续当前页面的蓝色渐变头部 + 白色内容区结构
- 头部展示：
  - 成员姓名
  - 性别 / 年龄
  - 档案号
- 内容区展示当前成员的记录摘要和就诊记录子卡片列表

4. 就诊记录子卡片

- 每次就诊记录为一张独立子卡片
- 卡片头部信息包含：
  - 第几次就诊
  - 就诊日期
  - 本次检查摘要
  - 当前状态文案，如 `已展开`、`点击查看`
- 默认展开最近一次记录，其余记录默认收起
- 点击收起项时：
  - 切换当前展开项为该记录
  - 原已展开项自动收起
- 同一成员同一时刻只允许展开 1 条记录

5. 记录展开详情区

- 展开区位于对应就诊记录子卡片内部
- 复用当前档案页的展示语义：
  - 左右眼图例
  - `眼轴 / 屈光度` 分段切换
  - 走势图或指标展示区域
- 默认先展示 `眼轴`
- 切换到 `屈光度` 时仅切换当前展开记录的内容，不影响其他成员或其他记录

6. 新增成员入口

- `+ 新增` 既出现在成员切换条中，也在当前内容区底部提供独立卡片提示
- 新增后生成第 3 位成员档案，默认处于 `0 次记录` 状态
- `0 次记录` 成员的主卡片仅展示基础身份信息和空状态说明，不显示记录列表

7. 底部预约浮层

- `去预约` 按钮固定在页面底部
- 预约浮层始终显示，不随当前成员是否有记录而消失
- 页面滚动时，记录区域在浮层上方滚动，避免按钮被内容顶走

**状态设计**

1. 成员 A：2 次记录

- 成员切换条显示 `2次`
- 主卡片内容区展示 2 条就诊记录子卡片
- 默认展开最近一次复查记录
- 另一条记录以收起态显示

2. 成员 B：1 次记录

- 成员切换条显示 `1次`
- 主卡片只展示 1 条记录
- 该条记录默认展开，减少额外点击

3. 成员 C：0 次记录

- 通过 `+ 新增` 进入或新增后出现在成员切换条中
- 主卡片展示：
  - 姓名 / 性别 / 年龄占位信息
  - `暂无检查记录` 文案
  - 引导用户直接预约或继续补充信息
- 底部 `去预约` 按钮仍然固定可见

**交互规则**

1. 成员切换

- 点击成员切换条中的任一成员后，替换当前主卡片数据
- 切换成员时，该成员优先展开其默认记录：
  - 多条记录时展开最近一次
  - 单条记录时默认展开该条
  - 无记录时展示空状态

2. 记录展开

- 记录子卡片头部整体可点击
- 点击未展开记录时展开该记录，并收起当前已展开记录
- 点击当前已展开记录时不做二次收起，避免用户丢失上下文

3. 指标切换

- `眼轴 / 屈光度` 切换只作用于当前展开记录
- 切换状态跟随记录本身保存，切换成员后可重置为默认 `眼轴`

4. 预约入口

- 点击 `去预约` 不要求本次接入真实预约流程
- 按钮保持为展示态或占位交互即可，但视觉上必须始终存在

5. 新增成员

- `+ 新增` 为展示态入口即可
- 本次不要求实现真实表单或成员创建流程
- 新增成员状态通过本地 mock 数据预置第 3 位成员表达

**数据策略**

本次仅使用本地 mock 数据，不接真实接口，不引入全局 store。

建议在 `RecordsModule.tsx` 内声明页面所需数据结构，示意如下：

```ts
type RecordMetricTab = "眼轴" | "屈光度";

type RecordMetricSnapshot = {
  rightLabel: string;
  rightValue: string;
  leftLabel: string;
  leftValue: string;
  conclusion?: string;
};

type VisitRecord = {
  id: string;
  title: string;
  date: string;
  summary: string;
  defaultExpanded?: boolean;
  metricTabDefault?: RecordMetricTab;
  axial: RecordMetricSnapshot;
  refraction: RecordMetricSnapshot;
};

type ArchiveProfile = {
  id: string;
  name: string;
  gender: string;
  age: string;
  archiveNo: string;
  visitCount: number;
  emptyHint?: string;
  visits: VisitRecord[];
  isAddEntry?: boolean;
};
```

Mock 约定：

- `周末`：`visitCount = 2`
- `安安`：`visitCount = 1`
- 第 3 位成员：`visitCount = 0`
- 每条记录提供 `眼轴` 和 `屈光度` 两套展示数据

**视觉设计**

- 延续当前 `RecordsModule` 的浅蓝背景、蓝色渐变头卡、白色圆角内容卡样式
- 人员切换条使用小卡片形式，不改为深色标签栏
- 就诊记录子卡片保持白底、浅描边、轻阴影，强调层次但不抢主卡片视觉重心
- 展开详情区不再新增复杂图表组件，优先以当前卡片式图例和摘要指标模拟趋势感
- 预约浮层继续使用高圆角蓝色主按钮，保持与当前页面一致的行动语义

**影响文件**

- 修改 `src/pages/miniprogram/components/RecordsModule.tsx`

如需把档案页状态与外层页面联动，再视实现情况决定是否轻微调整：

- 可选修改 `src/pages/miniprogram/index.tsx`

**不做事项**

- 不接入真实档案接口
- 不接入真实预约流程
- 不实现新增成员表单或编辑能力
- 不抽离独立路由或全局状态管理
- 不新增高复杂度图表库

**验证方式**

- 页面视觉检查：
  - 成员切换条可见且区分 2 次、1 次、0 次状态
  - `去预约` 始终固定在页面底部
  - 多次记录成员可切换查看不同记录详情
- 代码验证：
  - 确认 `RecordsModule.tsx` 无 TypeScript / JSX 诊断错误
  - 确认页面在原型容器中无明显布局溢出或滚动冲突
